

\section{Probabilidade Condicional, Independência e Teorema de Bayes}


\subsection{Axiomas de Kolmogorov}

Para continuarmos nosso estudo precisamos definir um espaço de probabilidade, representado por três elementos: $(\Omega, \mathcal{F}, P)$. Nesse modelo, $\Omega$ é o espaço amostral , $\mathcal{F}$ representa os eventos que podemos calcular e $P$ é a própria medida de probabilidade. Os seguintes axiomas foram criadas por Andrey Kolmogorov em 1933 \cite{zabala_fe}:


\begin{axioma}
	Para qualquer evento $A \in F$ , a probabilidade é um valor não negativo:
	\begin{align*}
		P(A) \geq 0, P(A) \in \mathbb{R}
	\end{align*}
\end{axioma}

\begin{axioma}
	A probabilidade de todo o espaço amostral $\Omega$  é igual a um:
	\begin{align*}
		P(\Omega) = 1
	\end{align*}
\end{axioma}

\begin{axioma}\label{axioma_3_kolmogorov}
	Se $A_1, A_2, \dots, A_k$ são conjuntos disjuntos, então:
	\begin{align*}
		P(A_1 \cup A_2 \cup \dots \cup A_k) = P(A_1) + P(A_2) + \dots + P(A_k)
	\end{align*}
	ou
	\begin{align*}
		P\left(\bigcup_{k=1}^{\infty} A_i\right) = \sum_{k=1}^{\infty} P(A_i)
	\end{align*}
\end{axioma}




\subsection{Probabilidade condicional}


Na análise de eventos aleatórios, a imprevisibilidade dos resultados é inerente ao processo. A obtenção de informações parciais prévias pode alterar a estimativa de probabilidade da ocorrência de um determinado evento. Considere, por exemplo, o cálculo da probabilidade de se obter uma soma igual a $8$ no lançamento de dois dados não viciados. A probabilidade a priori desse evento é de $\frac{5}{36}$. Entretanto, ao realizar o lançamento de um dos dados primeiramente, adquire-se uma informação que condiciona a expectativa do evento final. Se o primeiro dado apresentar a face $5$, será estritamente necessário obter a face $3$ no segundo lançamento, cuja probabilidade isolada é de $\frac{1}{6}$. Portanto, dado que o resultado inicial foi $5$, a probabilidade de obter uma soma igual a $8$ é atualizada de $\frac{5}{36}$ para $\frac{1}{6}$. Em contrapartida, assuma que o primeiro lançamento resulte na face $1$. Nesse contexto, torna-se impossível atingir a soma $8$, independentemente do valor obtido no lançamento subsequente. Por conseguinte, sob essa condição, a probabilidade de se obter o resultado de $8$ torna-se nula. \cite{Dougas_2002}

A probabilidade condicional nos diz que a probabilidade do evento $A$ ocorrer dado que $B$ é definida por:

\begin{equation} \label{probabilidade_condicional}
	P(A\mid B) = \frac{P(A\cap B)}{P(B)} , P(B) \neq 0
\end{equation}


\begin{exemplo}
Tomemos o caso descrito anteriormente:
\begin{itemize}
	\item[] $A$: soma 8 no lançamento de um par de dados;
	\item[] $B$: obter 5 no primeiro dado.
\end{itemize}

Calcular $P(A\mid B)$, temos as probabilidades: 
\begin{itemize}
	\item[] $P(B) = \frac{1}{6}$
	\item[] $P(A \cap  B) \text{ só pode ocorrer se obtivermos \{5,3\}}= \frac{1}{36}$
\end{itemize}

Assim temos: 

\begin{align*}
	P(A\mid B) = \frac{\frac{1}{36}}{\frac{1}{6}} = \frac{1}{6}
\end{align*}


Segue a representação matricial do problema: 
\begin{itemize}
	\item \textcolor{blue}{\textbf{Azul}}: Representa o \textbf{Evento B}.
	\item \textcolor{green}{\textbf{Verde}}: Representa os casos do \textbf{Evento A} que ocorrem fora do Evento B.
	\item \textcolor{red}{\textbf{Vermelho}}: Representa a \textbf{Interseção}.
\end{itemize}

	(2,1) & (2,2) & (2,3) & (2,4) & (2,5) & \color{green}{(2,6)} \\

\end{exemplo}

\subsection{Regra do Produto}


A Regra do Produto permite expressar a probabilidade da ocorrência simultânea de diversos eventos a partir do valor de cada probabilidade condicional de cada um deles dados os eventos anteriores.

A equação \ref{probabilidade_condicional}, resulta no teorema do produto, ou probabilidade do evento intersecção.


\begin{teorema}[Teorema do Produto]
	Para dois eventos:
	\begin{align*}
		P(A \cap B) = P(A) \cdot P(B\mid A) = P(B) \cdot P(A\mid B)
	\end{align*}
	Para tês eventos:
	\begin{align*}
	P(A \cap B \cap C) = P(A) \cdot P(B\mid A) \cdot P(C \mid A\cap B)
	\end{align*}
	Forma geral:
	\begin{equation}\label{teorema_produto}
	P\left(\bigcap_{i=1}^{k} A_i\right) = P(A_1) \cdot P(A_2 | A_1) \cdot P(A_3 | A_1 \cap A_2) \cdots P(A_k | A_1 \cap \cdots \cap A_{k-1})
	\end{equation}
\end{teorema}



\subsection{Lei da Probabilidade Total}


Dizemos que $A_1, A_2, \cdots , \in F$ formam uma partição de $\Omega$, se são disjuntos e $\cup_k B = \Omega$. Partições são particularmente úteis em contextos onde determinado aspecto divide os resultados possíveis em casos, e é possível expressar determinadas relações em cada um desses casos separadamente.\cite{Leonardo2026}


\begin{teorema}[Teorema da Probabilidade Total]
	Seja uma sequência enumerável de eventos aleatórios $A_1,A_2, \dots ,A_k$, formando uma partição de $\Omega$. Como as intersecções $A_i \cap B$ são mutuamente excludentes, então de \ref{axioma_3_kolmogorov} temos:
	
	\begin{align*}
		P(B) = \sum_{i=1}^{k} P(A_i \cap B)
	\end{align*}
	
	Aplicando \ref{teorema_produto} temos :
	
	\begin{equation}\label{teorema_probabilidade_total}
		P(B) = \sum_{i} P(A_i) \cdot P(B \mid A_i)
	\end{equation}
	
\end{teorema}

\subsection{Teorema de Bayes}

Do teorema da probabilidade total, podemos desenvolver o Teorema de Bayes.Tomando \ref{probabilidade_condicional}, podemos calcular a probabilidade de $A_i$, dada a ocorrência de $B$ por:

\begin{align*}
	P(A_i\mid B) = \frac{P(A_i\cap B)}{P(B)}
\end{align*}
\\
Aplicando o Teorema da probabilidade total \ref{teorema_probabilidade_total} no denominador e a regra do produto \ref{teorema_produto} no númerador, temos: 



\begin{teorema}[Teorema de Bayes]
\begin{equation}\label{Teorema_bayes}
	P(A_i\mid B) = \frac{P(A_i) \cdot P(B\mid A_i)}{\sum_{j} P(A_i) \cdot P(B\mid A_i)}
\end{equation}
\end{teorema}

O teorema de Bayes é útil quando conhecemos as probabilidades condicionais de $B$ dado $A_i$, mas não diretamente a probabilidade de $B$ \cite{zabala_fe}.


\begin{exemplo}
	Suponha uma caixa com três moedas, uma equilibrada e duas com duas faces cara. A probabilidade condicional de a moeda sorteada ter sido a equilibrada pode ser calculada. Para isso pode-se definir $A_1$: "a moeda retirada é equilibrada", $A_1 = \frac{1}{3}$, $A_2$: "a moeda retirada tem duas caras",$A_2 = \frac{2}{3}$  e $B$: "o resultado final é cara", $B = \frac{5}{6}$ e aplicar a regra de Bayes, resultando em:

\begin{align*}
	P(A_1\mid B) = \frac{P(A_1) \cdot P(B\mid A_1)}{P(A_1) \cdot P(B\mid A_1) + P(A_2) \cdot P(B \mid A_2)} = \frac{\frac{1}{3} \cdot \frac{1}{2}}{\frac{1}{3} \cdot \frac{1}{2} + \frac{2}{3} \cdot 1} = \frac{1}{5}
\end{align*}
	
	
	
\end{exemplo}




\subsection{Independência de Eventos}


Dois eventos são ditos independentes quando a realização de um não altera o espaço amostral ou a probabilidade de ocorrência do outro. Para ilustrar essa propriedade, considere o lançamento sequencial de dois dados ideais. Definem-se os eventos:
\begin{itemize}
	\item[] $A$: obtenção da face $3$ no primeiro ensaio;
	\item[] $B$: obtenção da face $5$ no segundo ensaio.
\end{itemize}
As probabilidades de cada evento são $P(A) = P(B) = \frac{1}{6}$. Dado que os lançamentos são independentes, a informação sobre a ocorrência de $B$ não afeta a probabilidade de $A$, o que implica que $P(A \mid B) = P(A) = \frac{1}{6}$.


\begin{definicao}[Eventos Independentes]
	Dois eventos $A$ e $B$ em um espaço amostral são considerados independentes se a ocorrência de um não afeta a probabilidade de ocorrência do outro. Em termos de probabilidade condicional, assumindo que $P(B) > 0$, dizemos que $A$ é independente de $B$ se:
	$$P(A \mid B) = P(A)$$

\end{definicao}


\begin{exemplo}
Temos um servidor de arquivos, com taxa de falha de $0,01$, e um servidor de backup como réplica do servidor principal, com taxa de falha de $0,01$, qual a probabilidade de ambos falharem?\\
Se os dois sistemas são independentes, basta multiplicar as probabilidades:
\begin{align*}
	P(A \text{ e } B) = P(A) \cdot P(B) = 0,01 \cdot 0,01 = 0,0001
\end{align*}
\end{exemplo}

\subsubsection{Filtro Anti-Spam}


Este experimento busca avaliar o desempenho de um classificador Binário, um filtro anti-spam. Para simular o cenário, utilizamos uma simulação estocástica de $n = 10.000$ observações (e-mails). O experimento assume os seguintes parâmetros e características do filtro: 

\begin{itemize}
	\item Prevalência -- $P(S)$ : probabilidade de um e-mail ser classificado como spam é de $20\%$;
	\item Sensibilidade --  ($P(F^+|S)$): taxa de verdadeiros positivos é de $95\%$;
	\item Especifidade -- $P(F^-|\neg S))$  taxa de verdadeiros negativos é de $90\%$, logo a taxa de falsos positivos é de $P(F^+|\neg S) = 90\%$ 

\end{itemize}

A geração dos dados reais seguiu uma distribuição Binomial (via função rbinom), alocando cada e-mail nas categorias "Spam" ou "Não Spam" com base na prevalência estipulada. Em seguida, a classificação do filtro foi aplicada condicionalmente à classe real de cada e-mail, respeitando as taxas de sensibilidade e especificidade predefinidas. Assim gerando a seguinte matriz de confusão, com a quantidade de cada categoria.

\begin{table}[ht]
	\centering
	\begin{tabular}{rrr}
		\hline
		& Não Spam & Spam \\ 
		\hline
		Não Spam & 7202 & 812 \\ 
		Spam &  83 & 1903 \\ 
		\hline
	\end{tabular}
	\caption{Matriz de Confusão do Filtro de Spam} 
	\label{tab:matriz_confusao}
\end{table}



Utilizando o (teorema da probabilidade total) \ref{teorema_probabilidade_total}, temos que a probabilidade do filtro classificar qualquer e-email como spam  $P(F^+)$ é dada por:


\begin{align*}
	P(F^+) = P(S) \cdot P(F^+|S) + P(\neg S) \cdot P(F^+|\neg S) = 0.27 = 27 \%
\end{align*}

A partir disto, utilizando o Teorema de Bayes \ref{Teorema_bayes} podemos calcular as métricas de performance do filtro. 

\begin{itemize}
	
	\item Valor Preditivo Positivo $P(S|F^+)$: A probabilidade de o e-mail ser spam, dado que o filtro o classificou como tal
	$$P(S|F^+) = \frac{P(S) \cdot P(F^+|S)}{P(F^+)} = 0,7037 = 70,37 \%$$
	
	\item Valor Preditivo Negativo $P(\neg S|F^-)$: A probabilidade de o e-mail ser legítimo, dado que o filtro o classificou como não spam.
	
		$$P(\neg S|F^-) = \frac{P(\neg S) \cdot P(F^-|\neg S)}{1 - P(F^+)} = 0,9863 = 98,63 \%$$
	
	
\end{itemize}

O valor predito positivo $P(S|F^+)$, ou seja, a chance do e-mail ser realemnte um spam, ficou em $70,37\%$ o que é baixo para um filtro anti spam, embora a especificidade do filtro ser de $90\%$, a baixa prevalência original de spam $20\%$ faz com que o volume de e-mails legítimos classificados como spam seja muito grande.

O valor predito negativo $P(\neg S|F^-)$, ou seja, a chance do e-mail ser classificado como legitimo, sendo que ele é um spam, é de $98,63\%$, o que é adequado para um filtro de spam.


O código-fonte utilizado para esta análise encontra-se disponível no Código \ref{cod filtro anti spam}.

\begin{lstlisting}[language=R, caption={Filtro Anti-Spam.}, label=cod filtro anti spam]

# ==================================================
# Filtro Anti-Spam
# ==================================================
cat("\n========== SEÇÃO 1: FILTRO ANTI-SPAM ==========\n")

# Definição dos eventos:
# S: email é spam (S) ou não (N)
# F: filtro classifica como spam (positivo) ou não (negativo)

# Parâmetros iniciais:
prev_spam <- 0.20     # Prevalência de spam
sens_filtro <- 0.95   # Sensibilidade: P(F+|S)
espec_filtro <- 0.90  # Especificidade: P(F-|N)

# Simulação de 10000 emails
n <- 10000
spam_real <- rbinom(n, 1, prev_spam)

# Aplicação do filtro (simulação condicional)
# Se for spam, probabilidade de classificar como spam = sens_filtro
# Se não for spam, probabilidade de classificar como spam = 1 - espec_filtro
filtro_pos <- rep(NA, n)
for (i in 1:n) {
	if (spam_real[i] == 1) {
		filtro_pos[i] <- rbinom(1, 1, sens_filtro)
	} else {
		filtro_pos[i] <- rbinom(1, 1, 1 - espec_filtro)
	}
}

# Matriz de confusão
confusion <- table(Real = ifelse(spam_real, "Spam", "Não Spam"),
Filtro = ifelse(filtro_pos, "Spam", "Não Spam"))
cat("Matriz de Confusão:\n")
print(confusion)

# exportar matriz de confusão para latex
print(xtable(confusion, 
caption = "Matriz de Confusão do Filtro de Spam", 
label = "tab:matriz_confusao"), 
type = "latex", 
include.rownames = TRUE)


# Probabilidade total de filtro positivo:
P_F_pos <- prev_spam * sens_filtro + (1 - prev_spam) * (1 - espec_filtro)
cat(sprintf("P(Filtro positivo) = %.4f\n", P_F_pos))

# Teorema de Bayes: Valor Preditivo Positivo (VPP) e Negativo (VPN)
VPP <- (prev_spam * sens_filtro) / P_F_pos
VPN <- ((1 - prev_spam) * espec_filtro) / (1 - P_F_pos)
cat(sprintf("VPP (P(Spam|F+)) = %.4f\n", VPP))
cat(sprintf("VPN (P(Não Spam|F-)) = %.4f\n", VPN))

# Gráfico: Barras da classificação real vs predita
df1 <- data.frame(
Real = c("Spam", "Não Spam"),
Predito_Spam = c(confusion[1,2]/sum(confusion[1,]), confusion[2,2]/sum(confusion[2,])),
Predito_NaoSpam = c(confusion[1,1]/sum(confusion[1,]), confusion[2,1]/sum(confusion[2,]))
)
df1_long <- pivot_longer(df1, cols = starts_with("Predito"),
names_to = "Predito", values_to = "Proporcao")

g1 <- ggplot(df1_long, aes(x = Real, y = Proporcao, fill = Predito)) +
geom_bar(stat = "identity", position = "dodge") +
labs(title = "Filtro Anti-Spam: Proporções de Classificação",
y = "Proporção", fill = "Classificação do Filtro") +
theme_minimal()
print(g1)

\end{lstlisting}



\subsubsection{Sistema de Alarmes}

A avaliação da confiabilidade de sistemas de segurança físicos exige a compreensão das variáveis que podem levar ao acionamento indevido ou à falha de detecção. O problema aqui modelado consiste em um sistema de alarme de segurança sujeito a dois eventos independentes: uma intrusão real e uma falha elétrica. 

O objetivo é mensurar a confiabilidade do disparo do alarme, ou seja, determinar qual é a probabilidade de estar ocorrendo uma intrusão real dado que o alarme disparou. Esse cálculo é importante, pois falsos positivos geram custos operacionais e reduzem a confiança no sistema, enquanto falsos negativos comprometem a segurança.

\begin{itemize}
	\item $I$: Ocorrência de intrusão (onde $I^c$ representa a não intrusão).
	\item $F$: Ocorrência de falha elétrica (onde $F^c$ representa a operação normal de energia).
	\item $A$: Disparo do alarme.
\end{itemize}

Assumimos as seguintes probabilidades a priori:
\begin{itemize}
	\item Probabilidade de intrusão: $P(I) = 0.01$ (1\%)
	\item Probabilidade de falha elétrica: $P(F) = 0.05$ (5\%)
\end{itemize}


O comportamento do alarme é estocástico e depende do estado de intrusão e de falha elétrica. As probabilidades condicionais de disparo do alarme ($A$) sob diferentes circunstâncias são:
\begin{itemize}
	\item $P(A|I, F) = 0.99$ (Disparo em caso de intrusão simultânea a uma falha).
	\item $P(A|I, F^c) = 0.95$ (Disparo em caso de intrusão sem falha elétrica).
	\item $P(A|I^c, F) = 0.10$ (Probabilidade de falso positivo induzido apenas pela falha elétrica).
	\item $P(A|I^c, F^c) = 0.001$ (Falso positivo por motivos aleatórios).
\end{itemize}


Dado as probabilidades de intrusão e de falha elétrica, simulamos $10000$ situações hipotéticas, usando a distribuição binomial, assim gerando: 


\begin{table}[ht]
	\centering
	\begin{tabular}{rrr}
		\hline
		& Intrusão & Não Intrusão \\ 
		\hline
		Intrusão &   6 &  85 \\ 
		Não Intrusão & 475 & 9434 \\ 
		\hline
	\end{tabular}
	\caption{Matriz de Confusão do Sistema de alarme} 
	\label{tab:matriz_confusao_sistema_de_alarme}
\end{table}



Determinar a probabilidade do alarme disparar, $P(A)$, utilizando o Teorema da Probabilidade Total \ref{teorema_probabilidade_total}.

\begin{align*}
	P(A) = P(A|I,F) \cdot P(I) \cdot P(F) + P(A|I,F^c) \cdot P(I) \cdot P(F^c) + \\
	 P(A|I^c,F) \cdot P(I^c) \cdot P(F) + P(A|I^c,F^c) \cdot P(I^c) \cdot P(F^c)
\end{align*}


Substituindo pelos parâmetros conhecidos:
\begin{align*}
	P(A) = (0.99 \cdot 0.01 \cdot 0.05) + (0.95 \cdot 0.01 \cdot 0.95)+ \\
	 (0.10 \cdot 0.99 \cdot 0.05) + (0.001 \cdot 0.99 \cdot 0.95)
\end{align*}
\begin{align*}
	P(A) \approx 0.0154 \quad (1.54\%)
\end{align*}


Para encontrar a probabilidade de ter ocorrido uma intrusão dado que o alarme tocou, $P(I|A)$, precisamos primeiro da probabilidade de o alarme tocar dado que houve uma intrusão, $P(A|I)$. Aplicando novamente a probabilidade total \ref{teorema_probabilidade_total} sobre as condições de falha:

\begin{align*}
	P(A|I) = P(A|I,F) \cdot P(F) + P(A|I,F^c) \cdot P(F^c)
\end{align*}
\begin{align*}
	P(A|I) = (0.99 \cdot 0.05) + (0.95 \cdot 0.95) = 0.9520
\end{align*}

Aplicando o Teorema de Bayes \ref{Teorema_bayes}:
\begin{align*}
	P(I|A) = \frac{P(I) \times P(A|I)}{P(A)}
\end{align*}
\begin{align*}
	P(I|A) = \frac{0.01 \times 0.9520}{0.0154} \approx 0.6178 \quad (61.2\%)
\end{align*}


Os resultados indicam que, apesar da alta taxa de acerto do alarme em condições de intrusão $95.2\%$, a baixa probabilidade prévia do evento de intrusão aliada à influência das falhas elétricas faz com que a confiabilidade posterior seja moderada. Em termos práticos, interpreta-se que, quando o alarme dispara, a probabilidade real de haver uma intrusão no local é de $61.8\%$. Os $38.2\%$ restantes representam alarmes falsos, sendo predominantemente impulsionados pelas falhas elétricas na rede.

\begin{lstlisting}[language=R, caption={Sistema de alarme.}, label=cod sistema de alarme]
# ==================================================
# Sistema de Alarme
# ==================================================
cat("========== SEÇÃO 2: SISTEMA DE ALARMES ==========\n")

# Definição dos eventos:
# I: intrusão (I) ou não (NI)
# F: falha elétrica (F) ou não (NF)
# A: alarme dispara (A) ou não (NA)

# Parâmetros:
P_I <- 0.01    # Probabilidade de intrusão
P_F <- 0.05    # Probabilidade de falha elétrica

# Probabilidades condicionais do alarme:
# P(A|I, F) = 0.99 (quase certo disparar se ambos)
# P(A|I, NF) = 0.95
# P(A|NI, F) = 0.10 (falso positivo por falha)
# P(A|NI, NF) = 0.001 (falso positivo raro)

# Simulação de 10000 situações
n2 <- 10000
intrusao <- rbinom(n2, 1, P_I)
falha <- rbinom(n2, 1, P_F)

alarme <- rep(NA, n2)
for (i in 1:n2) {

\subsection{Análise do Classificador Probabilístico Naive Bayes}

Nesta seção, avaliamos o desempenho e o comportamento do espaço de decisão de um classificador probabilístico Bayesiano Ingênuo (\textit{Naive Bayes}). O modelo foi treinado com um conjunto de dados simulado contendo 500 amostras, distribuídas em três classes ($C_1$, $C_2$ e $C_3$) e caracterizadas por dois atributos contínuos ($X_1$ e $X_2$). 

As probabilidades a priori das classes foram definidas empíricamente como $P(C_1) = 0.3$, $P(C_2) = 0.5$ e $P(C_3) = 0.2$. O modelo assume que, dado a classe, os atributos são condicionalmente independentes e seguem distribuições normais bivariadas. A probabilidade a posteriori para a classificação de um novo vetor de características $\mathbf{x} = (x_1, x_2)$ é dada pela regra de Bayes:

\begin{align*}
	P(C_k \mid x_1, x_2) = \frac{P(C_k) \cdot f(x_1 \mid C_k) \cdot f(x_2 \mid C_k)}{\sum_{j=1}^{3} P(C_j) \cdot f(x_1 \mid C_j) \cdot f(x_2 \mid C_j)}
\end{align*}

Onde $f(x_i \mid C_k)$ representa a função de densidade de probabilidade normal para o atributo $i$ na classe $k$.


Para validar a capacidade preditiva do modelo, submeteu-se a instância arbitrária localizada nas coordenadas $(2.0, 1.0)$ ao classificador. Utilizando os parâmetros extraídos do treinamento, o algoritmo calculou as seguintes probabilidades a posteriori normalizadas:

\begin{itemize}
	\item $P(C_1 \mid \mathbf{x}) \approx 0.229$ ($22,9\%$)
	\item $P(C_2 \mid \mathbf{x}) \approx 0.745$ ($74,5\%$)
	\item $P(C_3 \mid \mathbf{x}) \approx 0.026$ ($2,6\%$)
\end{itemize}

Aplicando a regra de decisão \textit{Maximum A Posteriori} (MAP), o modelo atribui a instância à classe que maximiza a probabilidade. Conforme demonstrado pelos resultados, o ponto $(2.0, 1.0)$ foi corretamente predito como pertencente à classe \textbf{$C_2$}, evidenciando forte confiança na predição em virtude da alta margem probabilística em relação às demais classes.


Para uma compreensão espacial de como o algoritmo discrimina as classes, o espaço amostral foi mapeado por meio de uma grade (\textit{grid}) de coordenadas. A Figura \ref{fig:fronteiras_bayes} ilustra o diagrama de dispersão das amostras sobreposto às regiões de decisão do classificador probabilístico.

\begin{lstlisting}[language=R, caption={Classificador Naive Bayes.}, label=cod cassificador_bayes]

cat("========== CLASSIFICAÇÃO PROBABILÍSTICA ==========\n")

# Parâmetros: Médias e desvios para cada classe (atributos X1 e X2)
classes <- c("C1", "C2", "C3")
prior <- c(0.3, 0.5, 0.2)  # Probabilidades a priori

# Parâmetros das distribuições condicionais (assumindo normal)
mu <- matrix(c(0, 0,  # C1: X1, X2
3, 2,  # C2: X1, X2
5, -1), nrow=3, byrow=TRUE)  # C3: X1, X2
sigma <- matrix(c(1.5, 1.5, 1.5, 1.5, 1.5, 1.5), nrow=3, byrow=TRUE)  # desvios padrão

# Simular 500 pontos de treinamento
n_train <- 500
classes_verdade <- sample(classes, n_train, replace = TRUE, prob = prior)
X1 <- rep(NA, n_train)
X2 <- rep(NA, n_train)
for (i in 1:n_train) {
	idx <- which(classes == classes_verdade[i])
	X1[i] <- rnorm(1, mu[idx,1], sigma[idx,1])
	X2[i] <- rnorm(1, mu[idx,2], sigma[idx,2])
}
dados <- data.frame(Classe = classes_verdade, X1, X2)

# Função de densidade normal
dnorm2d <- function(x1, x2, mu1, mu2, s1, s2) {
	exp(-0.5*(((x1-mu1)/s1)^2 + ((x2-mu2)/s2)^2)) / (2*pi*s1*s2)
}

# Classificar um novo ponto
novo <- c(2, 1)  # (X1, X2)
post_probs <- rep(NA, 3)
for (k in 1:3) {
	likelihood <- dnorm2d(novo[1], novo[2], mu[k,1], mu[k,2], sigma[k,1], sigma[k,2])
	post_probs[k] <- prior[k] * likelihood
}
post_probs <- post_probs / sum(post_probs)

cat(sprintf("Ponto (%.1f, %.1f) - Probabilidades Posteriores:\n", novo[1], novo[2]))
for (k in 1:3) {
	cat(sprintf("  %s: %.3f\n", classes[k], post_probs[k]))
}
cat(sprintf("Classe predita: %s\n", classes[which.max(post_probs)]))

# Criar grid de pontos
x1_range <- seq(min(X1)-1, max(X1)+1, length.out = 100)
x2_range <- seq(min(X2)-1, max(X2)+1, length.out = 100)
grid <- expand.grid(X1 = x1_range, X2 = x2_range)

# Calcular probabilidades para cada ponto do grid
grid_pred <- matrix(NA, nrow = nrow(grid), ncol = 3)
for (i in 1:nrow(grid)) {
	for (k in 1:3) {
		grid_pred[i,k] <- prior[k] * dnorm2d(grid$X1[i], grid$X2[i], mu[k,1], mu[k,2], sigma[k,1], sigma[k,2])
	}
	grid_pred[i,] <- grid_pred[i,] / sum(grid_pred[i,])
}
grid$Classe <- classes[apply(grid_pred, 1, which.max)]
grid$Prob <- apply(grid_pred, 1, max)


g3 <- ggplot() +
geom_tile(data = grid, aes(x = X1, y = X2, fill = Classe), alpha = 0.3) +
geom_contour(data = grid, aes(x = X1, y = X2, z = as.numeric(factor(Classe))), 
breaks = c(1.5, 2.5), color = "black", alpha = 0.5, inherit.aes = FALSE) +
geom_point(data = dados, aes(x = X1, y = X2, color = Classe), size = 2) +
labs(title = "Classificação Bayes Ingênuo com Fronteiras",
x = "Atributo X1",
y = "Atributo X2",
fill = "Região Predita",
color = "Classe Real") +
theme_minimal()

print(g3)

\subsection{Autenticação Biométrica}
\label{subsec:analise_biometrica}

A avaliação da eficácia e da confiabilidade de sistemas de autenticação biométrica fundamenta-se no estudo do comportamento das taxas de erro diante da variação do limiar operacional de decisão ($t$). As duas métricas primordiais para essa análise são a Taxa de Falsa Aceitação (\textit{False Acceptance Rate} -- FAR), que mensura a probabilidade de o sistema classificar incorretamente um impostor como usuário legítimo, e a Taxa de Falsa Rejeição (\textit{False Rejection Rate} -- FRR), que indica a probabilidade de um usuário legítimo ser indevidamente bloqueado.

Nesta seção, analisa-se o desempenho de um modelo biométrico simulado em ambiente computacional R, cujos escores de correspondência (\textit{matching scores}) revelam o comportamento do sistema sob condições controladas de separabilidade de classes.


A simulação considerou duas populações estatísticas independentes, contendo $n = 1000$ amostras cada, representando os escores de similaridade atribuídos a tentativas de acesso de usuários genuínos e de indivíduos impostores. Os escores foram modelados a partir de distribuições normais com os seguintes parâmetros operacionais:

\begin{itemize}
	\item \textbf{Escores Genuínos ($S_{\text{gen}}$):} $\mathcal{N}(\mu_{\text{gen}} = 0{,}9; \,\sigma_{\text{gen}} = 0{,}1)$
	\item \textbf{Escores Impostores ($S_{\text{imp}}$):} $\mathcal{N}(\mu_{\text{imp}} = 0{,}2; \,\sigma_{\text{imp}} = 0{,}1)$
\end{itemize}

O limiar de decisão $t$ foi avaliado de forma incremental ao longo do intervalo discreto $[0, 1]$ com passo de $0{,}01$. Para cada valor de $t$, as taxas FAR e FRR foram computadas de forma empírica através das seguintes relações de probabilidade:

\begin{align*}
	\text{FAR}(t) = P(S_{\text{imp}} > t) = \frac{1}{n}\sum_{i=1}^{n} \mathbb{I}(S_{\text{imp}, i} > t)
\end{align*}

\begin{align*}
	\text{FRR}(t) = P(S_{\text{gen}} \le t) = \frac{1}{n}\sum_{i=1}^{n} \mathbb{I}(S_{\text{gen}, i} \le t)
\end{align*}

Onde $\mathbb{I}(\cdot)$ representa a função indicadora, que assume valor 1 se a condição for verdadeira e 0 caso contrário.

O comportamento empírico observado na saída dos dados demonstra o trade-off clássico de sistemas classificadores lineares. A variação de $t$ acarreta deslocamentos inversamente proporcionais entre as taxas de erro:
\begin{itemize}
	\item \textbf{Caso de Baixo Limiar ($t \to 0$):} O critério de aceitação torna-se excessivamente permissivo. Como consequência, a FAR atinge seu patamar máximo ($\text{FAR} \approx 1{,}0$), permitindo o acesso de quase todos os impostores, ao passo que a FRR é minimizada ($\text{FRR} \approx 0{,}0$).
	\item \textbf{Caso de Alto Limiar ($t \to 1$):} O sistema torna-se estrito e altamente seguro. A FAR decai para $0{,}0$, inviabilizando fraudes, porém a taxa de rejeição de usuários legítimos (FRR) eleva-se drasticamente para próximo de $1{,}0$.
\end{itemize}

O ponto ótimo de operação é balizado pela \textit{Equal Error Rate} (EER) --- a Taxa de Erro Igualitário --- configurada no ponto de intersecção onde $\text{FAR}(t) = \text{FRR}(t)$. De acordo com o algoritmo de minimização aplicado sobre os resultados simulados, obteve-se:
\begin{itemize}
	\item \textbf{Limiar Ótimo Recomendado ($t$):} $0{,}52$
	\item \textbf{EER Aproximado:} $0$ ($0\%$)
\end{itemize}

A obtenção de uma EER nula, neste experimento decorre da completa separabilidade entre as funções de densidade de probabilidade geradas. Analisando, a distância entre as médias das distribuições é dada por $\Delta\mu = \mu_{\text{gen}} - \mu_{\text{imp}} = 0{,}9 - 0{,}2 = 0{,}7$. Dado que o desvio padrão de ambas as curvas é $\sigma = 0{,}1$, a distância correspondente entre os picos equivale a $7\sigma$.

A dinâmica de cruzamento das taxas descritas e o comportamento das curvas frente ao limiar calculado estão ilustrados na Figura~\ref{fig:curva_far_frr}.

	\caption{Curvas de variação das taxas FAR e FRR em função do limiar de decisão. A linha vertical tracejada demarca o ponto de equilíbrio operacional (EER) obtido em $t = 0{,}52$.}

n_genuino <- 1000
n_impostor <- 1000

scores_genuino <- rnorm(n_genuino, mean = 0.9, sd = 0.1)

\subsection{Diagnóstico Médico}

Nesta sessão, abordamos a avaliação do desempenho de testes diagnósticos sob a ótica da epidemiologia quantitativa.

Para ilustrar este comportamento, analisamos um teste diagnóstico configurado com os seguintes parâmetros fixos:
\begin{itemize}
	\item \textbf{Sensibilidade ($S$):} $99\%$, representando a probabilidade de o teste resultar positivo dado que o indivíduo possui a patologia.
	\item \textbf{Especificidade ($E$):} $95\%$, que indica a probabilidade de o teste resultar negativo dado que o indivíduo é saudável.
\end{itemize}

O comportamento do Valor Preditivo Positivo (VPP) e do Valor Preditivo Negativo (VPN) foi avaliado variando-se a prevalência da doença ($\phi$) em cinco cenários distintos: $1\%$, $5\%$, $10\%$, $20\%$ e $50\%$.


Com base no Teorema de Bayes \ref{Teorema_bayes}, a probabilidade pós-teste de um indivíduo estar verdadeiramente enfermo após um resultado positivo (VPP), ou de estar perfeitamente saudável após um resultado negativo (VPN), é expressa formalmente pelas seguintes equações:

\begin{align*}
	VPP = \frac{\phi \cdot S}{\phi \cdot S + (1 - \phi) \cdot (1 - E)}
\end{align*}

\begin{align*}
	VPN = \frac{(1 - \phi) \cdot E}{(1 - \phi) \cdot E + \phi \cdot (1 - S)}
\end{align*}

Para compreender a dinâmica das probabilidades, realizou-se uma simulação estocástica baseada em ensaios de Bernoulli, para uma amostra teórica de $N = 10.000$ indivíduos, considerando um cenário com prevalência moderada de $10\%$ ($\phi = 0{,}10$). \begin{lstlisting}[language=R, caption={Testes Diagnósticos.}, label=cod testes diagnosticos]
sens_teste <- 0.99  # Sensibilidade
espec_teste <- 0.95 # Especificidade

# Cenários de prevalência
prev <- c(0.01, 0.05, 0.10, 0.20, 0.50)

VPP_calc <- function(prev, sens, esp) {
	P_pos <- prev * sens + (1 - prev) * (1 - esp)
	VPP <- (prev * sens) / P_pos
	return(VPP)
}
VPN_calc <- function(prev, sens, esp) {
	P_neg <- prev * (1 - sens) + (1 - prev) * esp
	VPN <- ((1 - prev) * esp) / P_neg
	return(VPN)
}

vpps <- sapply(prev, VPP_calc, sens = sens_teste, esp = espec_teste)
vpns <- sapply(prev, VPN_calc, sens = sens_teste, esp = espec_teste)

# Simulação para um cenário específico (prevalência = 0.10)
n5 <- 10000
prev5 <- 0.10
doenca <- rbinom(n5, 1, prev5)
teste <- rep(NA, n5)
for (i in 1:n5) {
	if (doenca[i] == 1) {
		teste[i] <- rbinom(1, 1, sens_teste)
	} else {
		teste[i] <- rbinom(1, 1, 1 - espec_teste)
	}
}
# Matriz de confusão
conf5 <- table(Doença = ifelse(doenca, "Sim", "Não"),
Teste = ifelse(teste, "Positivo", "Negativo"))
cat("Matriz de Confusão (Prevalência 10%):\n")
print(conf5)

# exportar matriz de confusão para latex
print(xtable(conf5,

type = "latex", 
include.rownames = TRUE)
